# Dockerfile and docker-compose — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** A small multi-stage Alpine image and a `docker-compose.yml` that auto-builds it and runs the app on port 8000.

**Architecture:** Stage 1 (`node:22-alpine`) builds the frontend into `tether_ddns/static`; stage 2 (`python:3.12-alpine`) installs the package into a venv with `uv`; stage 3 (`python:3.12-alpine`) is a slim non-root runtime.

**Tech Stack:** Docker (multi-stage, Alpine), uv, uvicorn.

## Global Constraints

- Final image runs `python -m tether_ddns` on port 8000 as a non-root user.
- Config persisted via `TETHER_DDNS_CONFIG_PATH=/data/tether-ddns.json` on a volume.
- No new Python/JS dependencies; existing test suites unaffected.
- Vite `outDir` is `../tether_ddns/static`; frontend has `package-lock.json`; repo has `uv.lock`.

### Task 2: Multi-stage Dockerfile

**Files:**
- Create: `Dockerfile`

**Interfaces:**
- Produces an image whose `CMD` runs `python -m tether_ddns` on port 8000.

- [ ] **Step 1: Create `Dockerfile`**

```dockerfile
# syntax=docker/dockerfile:1

# --- Stage 1: build the frontend ---
FROM node:22-alpine AS frontend
WORKDIR /app/frontend
COPY frontend/package.json frontend/package-lock.json ./
RUN npm ci
COPY frontend/ ./
# Vite emits to ../tether_ddns/static (outDir), so provide the package dir.
COPY tether_ddns/ /app/tether_ddns/
RUN npm run build

# --- Stage 2: install the Python package into a venv ---
FROM python:3.12-alpine AS builder
RUN apk add --no-cache build-base gcc musl-dev libffi-dev \
    && pip install --no-cache-dir uv
WORKDIR /app
COPY pyproject.toml uv.lock README.md ./
COPY tether_ddns/ ./tether_ddns/
# Bring in the built static assets from stage 1.
COPY --from=frontend /app/tether_ddns/static ./tether_ddns/static
RUN python -m venv /opt/venv \
    && uv pip install --python /opt/venv/bin/python .

# --- Stage 3: slim runtime ---
FROM python:3.12-alpine AS runtime
RUN apk add --no-cache libstdc++ \
    && adduser -D -h /home/app app \
    && mkdir /data && chown app:app /data
COPY --from=builder /opt/venv /opt/venv
COPY --from=builder /app/tether_ddns /app/tether_ddns
ENV PATH=/opt/venv/bin:$PATH \
    TETHER_DDNS_CONFIG_PATH=/data/tether-ddns.json \
    PYTHONUNBUFFERED=1
WORKDIR /app
EXPOSE 8000
USER app
CMD ["python", "-m", "tether_ddns"]
```

- [ ] **Step 2: Build the image**

Run: `docker build -t tether-ddns:local .`
Expected: build completes successfully through all three stages.

If a musl wheel fails to compile in stage 2, add the missing `-dev` package to
the `apk add` line (e.g. `openssl-dev`, `cargo` for pydantic-core if a source
build is triggered) and rebuild. Record any additions.

- [ ] **Step 3: Smoke-test the container**

Run:
```bash

sleep 3
curl -sf http://localhost:8000/ | head -c 60
curl -sf http://localhost:8000/api/hooks | head -c 60
docker rm -f tether-smoke
```
Expected: the first curl returns SPA HTML (`<!doctype html>` / `<div id="root">`),
the second returns a JSON array. Record the image size from `docker images tether-ddns:local`.

### Task 3: docker-compose.yml

**Files:**
- Create: `docker-compose.yml`

- [ ] **Step 1: Create `docker-compose.yml`**

```yaml
services:
  tether-ddns:
    build: .
    image: tether-ddns:local
    ports:
      - "8000:8000"
    # For a DDNS tool, host networking can improve router/LAN reachability and
    # public-IP detection. To use it, comment out `ports` above and uncomment:
    # network_mode: host
    volumes:
      - tether-config:/data
    restart: unless-stopped

volumes:
  tether-config:
```

- [ ] **Step 2: Build and run via compose**

Run:
```bash
docker compose build
docker compose up -d
sleep 3
curl -sf http://localhost:8000/api/hooks | head -c 60
```
Expected: build succeeds; the endpoint returns JSON.

- [ ] **Step 3: Verify config persistence**

Run:
```bash
docker compose exec -T tether-ddns sh -c 'ls -la /data'
docker compose down
docker compose up -d
sleep 3
docker compose exec -T tether-ddns sh -c 'ls -la /data'
docker compose down
```
Expected: `/data` is writable by the app user and the config file persists
across the down/up cycle (named volume retained).
